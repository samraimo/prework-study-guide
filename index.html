HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The < h1 > is the opening tag and < /h1 > is the closing tag.
  • The < p > element represents a paragraph or block of text.
  • The < ul >, < ol >, and < li > elements represent unordered lists, ordered lists, and list items.

CSS

  • Creates the style for the webpage
  • A margin indicates how much space we want around the outside of an element.
  • Padding indicates how much space we want around the content inside an element.
  • Asterisk selector indicates that all elements will be affected

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git init- initialize an existing directory as a Git repository
  • git clone [url]- retrieve an entire repository from a hosted location via URL
  • git status- show modified files in working directory, staged for your next commit
  • git add [file]- add a file as it looks now to your next commit (stage)
  • git reset [file]- unstage a file while retaining the changes in working directory
  • git diff- diff of what is changed but not staged
  • git diff --staged- diff of what is staged but not yet commited
  • git commit -m “[descriptive message]”- commit your staged content as a new commit snapshot

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • The five most basic types of data are strings, numbers, booleans, undefined, and null.
  • A JavaScript function is a block of code designed to perform a particular task. A JavaScript function is executed when "something" invokes it (calls it).
